在 Python 中，像 `__all__`、`__name__`、`__init__` 这样**以双下划线开头和结尾**的标识符，有一套特殊的命名约定，通常被称为 **“dunder”**（即 double underscore 的缩写）**魔法方法**或**特殊变量**。它们不是普通的变量名，而是由 Python 语言本身**赋予特殊含义**的。

## 一、这种写法叫什么？

- **正式名称**：特殊方法（special methods）或特殊变量（special variables）。
    
- **口语称呼**：dunder methods / dunder variables（例如 `__init__` 读作 “dunder init”）。
    
- **特征**：名称前后各有两个下划线 `__`。
    

---

## 二、它代表什么意思？

- **对于方法**：它们定义了类的实例在某些特定操作（如初始化、加法、长度计算等）时的行为。例如：
    
    - `__init__(self, ...)`：对象初始化时自动调用。
        
    - `__add__(self, other)`：定义 `+` 运算符的行为。
        
- **对于变量**：它们通常包含 Python 解释器或模块的元信息。例如：
    
    - `__name__`：当前模块的名称（如果是主程序则为 `"__main__"`）。
        
    - `__file__`：当前模块所在的文件路径。
        
    - `__all__`：模块中当使用 `from module import *` 时，默认会被导入的名称列表。

## 三、常见双下划线内置变量/方法速查表

|名称|类型|作用|
|---|---|---|
|`__name__`|变量|当前模块名，`"__main__"` 或包名|
|`__file__`|变量|当前文件的路径|
|`__all__`|变量（列表）|控制 `from module import *` 的行为|
|`__init__(self)`|方法|类实例初始化|
|`__str__(self)`|方法|定义 `print(obj)` 或 `str(obj)` 的输出|
|`__repr__(self)`|方法|定义交互式环境下直接显示对象的信息|
|`__len__(self)`|方法|定义 `len(obj)` 的行为|
|`__getitem__(self, key)`|方法|定义 `obj[key]` 索引/键访问|
|`__call__(self, ...)`|方法|使实例可以像函数一样被调用|

## 四、一般什么时候写？

### 1. 定义类的“魔法方法”

当你想让类的实例支持某些内置行为时，你需要实现这些特殊方法。

python

class MyList:
    def __init__(self, items):
        self.items = items          # 初始化
    def __len__(self):
        return len(self.items)      # 支持 len()
    def __getitem__(self, index):
        return self.items[index]    # 支持索引访问

### 2. 在模块中定义 `__all__`

当你希望限制 `from module import *` 时暴露的接口，可以显式列出允许导入的名称。

python

# 模块 mymodule.py
__all__ = ['public_func', 'PublicClass']
def public_func(): pass
def _private_func(): pass          # 下划线开头是约定私有，但 import * 仍会导入，除非__all__限制
class PublicClass: pass

没有 `__all__` 时，`from mymodule import *` 会导入所有**不以单下划线开头的**名称。有了 `__all__` 后，只导入列表内的名称（即使是单下划线开头的也可以强行放入 `__all__` 导出）。

### 3. 使用模块的内置特殊变量

- ** `if __name__ == "__main__"` **：判断当前模块是直接运行还是被导入。
    
- **调试打印 `__file__` **：知道当前脚本所在位置。

## 五、`__all__` 在你截图中的具体作用

你的截图中：

python

# test_3.py
__all__ = ['add']
def add(x, y): ...
def sub(x, y): ...

然后如果另一个文件写：

python

from test_3 import *

只能使用 `add`，不能使用 `sub`。这就是 `__all__` 的限制作用。  
但是其他导入方式（如 `import test_3` 然后 `test_3.sub()` 或 `from test_3 import sub`）不受影响。
